Gobblet

Prototype Process:

  • Carefully chose the dimensions on the PIXI screen to be 1200px wide by 900px tall.
  • Created the art assets, and set up the grid-like board.
  • Set up the class architecture. The board is composed of Stacks, Stacks are composed of Pieces.
  • Loaded 24 pieces into 6 different stacks off of the board.
  • Enabled dragging functionality, and made pieces snap into place.
  • Made game loop. Player moves, then the opponent moves. So on and so forth until there is a winner.

Final Process:

  • Began implementation of Alpha-Beta pruning with the minimax algorithm.
  • Generated game tree with a depth of 3, and calculated (numerically) the best move for the opponent.
  • Had issues with propegating information about the best move back up the tree, so decided to break from this method of AI, and focused on an alternative, with every intention to revist it in the near future.
  • Started building a different AI that calculates moves based on a few rules. Move to a winning position if possible, Block the opponent from winning, and prioritize claiming center positions over outer positions.
